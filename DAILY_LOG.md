#### 1. 컴포넌트 아키텍처 구축
- **UI 컴포넌트 생성**
  - `Button.tsx` - 재사용 가능한 버튼 컴포넌트 (primary, secondary, danger, ghost)
  - `Input.tsx` - 폼 입력 필드 컴포넌트 (에러 처리, 라벨 지원)
  - `Card.tsx` - 카드 레이아웃 컴포넌트 (hover 효과, 패딩 옵션)

- **폴더 관련 컴포넌트**
  - `FolderCard.tsx` - 폴더 표시 카드 (편집/삭제 버튼)
  - `FolderForm.tsx` - 폴더 생성/편집 폼 (색상 선택기 포함)

#### 2. 폴더 관리 시스템 완성
- **페이지 구조**
  - `/folders` - 폴더 목록 페이지
  - `/folders/new` - 폴더 생성 페이지
  - `/folders/[id]/edit` - 폴더 편집 페이지
  - `/folders/[id]/delete` - 폴더 삭제 액션

- **기능 구현**
  - ✅ 폴더 CRUD (생성, 읽기, 수정, 삭제)
  - ✅ 사용자별 데이터 분리
  - ✅ 폴더별 태스크 개수 표시
  - ✅ 색상 선택 UI
  - ✅ 유효성 검사 (이름 길이 등)

#### 3. Git 브랜치 전략 설정

### 🔧 기술적 세부사항

#### 컴포넌트 설계 원칙
- **재사용성**: 모든 UI 컴포넌트는 다양한 상황에서 재사용 가능
- **일관성**: 동일한 스타일과 인터랙션 패턴
- **확장성**: 새로운 variant나 옵션 추가 용이

#### 데이터베이스 연동
- **Supabase RLS**: 사용자별 데이터 접근 제어
- **CASCADE 삭제**: 폴더 삭제 시 관련 태스크/노트도 함께 삭제
- **실시간 통계**: 각 폴더의 태스크 개수 실시간 계산
